# RC4 40-bit CUDA Brute Force 🚀

Сверхбыстрый инструмент для взлома PDF Revision 2 (40-битное RC4 шифрование) с использованием NVIDIA GPU.

## 🎯 Ключевые особенности

- ⚡ **Прямой перебор ключей** - пропускаем MD5 хэширование паролей
- 🚀 **GPU ускорение** - в ~1000 раз быстрее CPU
- 🎮 **Оптимизация для RTX 4070 Ti** - ~12 гигаключей в секунду
- ⏱️ **Быстрый взлом** - весь keyspace 2^40 за ~90 секунд на RTX 4070 Ti
- 💻 **CPU fallback** - работает и без GPU (медленнее)

## 💡 Как это работает

### Традиционный подход (медленный):
```
Пароль → MD5 хэш → Ключ → RC4 расшифровка → Проверка
```

### Наш подход (быстрый):
```
Прямой перебор всех 2^40 ключей → RC4 расшифровка → Проверка
```

**Преимущества:**
- ✅ Пропускаем MD5 хэширование (экономия времени)
- ✅ Перебираем ключи напрямую (меньше операций)
- ✅ Оптимизация под GPU (массивный параллелизм)

### Алгоритм:

1. **Инициализация:**
   - Загружаем U-поле в constant memory GPU
   - Загружаем padding string для проверки

2. **Параллельный перебор:**
   - Каждый CUDA thread проверяет свой диапазон ключей
   - Для каждого ключа:
     - Выполняем RC4 KSA (Key Scheduling)
     - Генерируем keystream (PRGA)
     - Расшифровываем U-поле
     - Сравниваем с padding string

3. **Результат:**
   - При совпадении - сохраняем ключ
   - Останавливаем все threads

## 🎮 Оптимизации

### 1. Constant Memory
```cuda
__constant__ uint8_t d_u_field[32];
__constant__ uint8_t d_padding[32];
```
Быстрый доступ для всех threads.

### 2. Inline RC4
```cuda
// Весь RC4 inline для минимизации overhead
#pragma unroll 256
for (int j = 0; j < STATE_SIZE; j++) {
    S[j] = j;
}
```


## 🐛 Решение проблем

### "CUDA error: no kernel image"

```bash
# Узнать compute capability
nvidia-smi --query-gpu=compute_cap --format=csv

# Пересобрать с правильной архитектурой
make cuda GPU_ARCH=sm_XX
```

### Медленная скорость

```bash
# Проверить температуру
nvidia-smi

# Увеличить power limit
sudo nvidia-smi -pl 300
```

### Out of memory

Уменьшить количество блоков в коде:
```cpp
const int blocks = prop.multiProcessorCount * 4;  // Вместо 8
```
