## 📊 Что происходит во время брутфорса

```
Пространство ключей: 2^40 = 1,099,511,627,776 ключей

Для каждого ключа (0x0000000000 до 0xFFFFFFFFFF):
  1. Выполнить RC4 KSA (Key Scheduling Algorithm)
  2. Сгенерировать keystream через PRGA
  3. Расшифровать U-поле
  4. Сравнить с padding string
  5. Если совпадает → КЛЮЧ НАЙДЕН! ✅
```

## 🔓 Использование найденного ключа

После нахождения ключа, создайте Python скрипт для расшифрования:

```python
#!/usr/bin/env python3
from rc4_pdf_revision2 import RC4

# Ключ, найденный брутфорсом (замените на реальный)
key = bytes.fromhex("A1B2C3D4E5")

# Читаем зашифрованный PDF
with open("encrypted.pdf", "rb") as f:
    encrypted_data = f.read()

# Расшифровываем
rc4 = RC4(key)
decrypted = rc4.decrypt(encrypted_data)

# Сохраняем
with open("decrypted.pdf", "wb") as f:
    f.write(decrypted)

print("✅ PDF расшифрован!")
```

## 📈 Почему это работает

### Слабости PDF Revision 2:

1. **40-битный ключ** - слишком короткий
   - Пространство: 2^40 = 1.1 триллиона
   - На GPU: взлом за минуты

2. **Устаревший RC4** - известные слабости
   - Статистические смещения
   - Предсказуемые паттерны

3. **Прямой перебор ключей** - эффективнее паролей
   - Пропускаем MD5 хэширование
   - Меньше операций на ключ

### Наша оптимизация:

- ✅ Constant memory для U-поля
- ✅ Inline RC4 для минимизации overhead
- ✅ Быстрая проверка первых байт
- ✅ Оптимальные параметры блоков/тредов
- ✅ Массивный параллелизм GPU
